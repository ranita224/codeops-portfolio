Q1.What is the difference between a class and an object?
Answer:- A class; is a blueprint or template that defines what properties and behaviors something will have.it doesn't hold real data itself.
An object; is an actual instance created from that blueprint, with its own real values in memory.
Q2.What does __init__ do, and when does it run?
Answer:- It is the constructor method,it runs automatically every time a new object is created from the class, and its job is to set up (initialize) that object's starting attributes.
Q3.What is self, and why is it the first parameter of every method?
Answer:- It refers to the specific object the method is being called on,it's how a method knows which object's data to work with, since a class can produce many different objects at once.
Q4.What problem does encapsulation solve in the Account class?
Answer:-Encapsulation protects an object's internal data from being changed in invalid or unsafe ways by hiding it (__balance) and only allowing changes through controlled methods (deposit, withdraw) that enforce rules
Q5.What's the difference between a public, protected, and private attribute?
Answer:- Public(self.owner): no underscore, accessible and changeable from anywhere, no restriction.
Protected (self._balance, single underscore): a convention signaling "internal use, please don't touch this from outside",but Python doesn't actually enforce it; it's just a hint to other programmers.
Private (self.__balance, double underscore): Python actively enforces this through name mangling, renaming it internally (e.g. to _Account__balance) so it can't be accessed directly from outside the class as self.__balance,you're expected to interact with it only through methods or properties defined inside the class.
Q6.Why use @property instead of a plain attribute for the balance?
Answer:-A plain attribute (self.balance) can be read and overwritten directly from outside with zero validation.Anyone could set it to anything. @property lets you expose a value for reading using the same simple syntax (account.balance, no parentheses), while giving you full control over what happens if someone tries to write to it.
